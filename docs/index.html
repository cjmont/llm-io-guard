🛡️ llm-io-guard

Offline, in-process I/O safety for LLM apps — Rust core, Node + edge/WASM.

Seguridad de E/S offline e in-process para apps con LLM — core en Rust, Node + edge/WASM.

Install

npm install llm-io-guard

Wrap one LLM call

import { Guard, piiRedact, promptInjection, secrets, piiRestore, secretLeak } from "llm-io-guard";

const guard = new Guard({
  input: [piiRedact(), promptInjection(), secrets()],
  output: [piiRestore(), secretLeak()],
  onViolation: "block",
});

const inp = await guard.scanInput(userText);     // PII → vault, no PII leaves your process
if (!inp.allowed) throw new Error("unsafe input");
const reply = await callYourLLM(inp.sanitized);  // send the redacted text
const out = await guard.scanOutput(reply, { vault: inp.vault }); // restore real PII
console.log(out.sanitized);

Defense-in-depth, not a guarantee. Heuristics have false negatives; this complements (not replaces) least privilege and separating instructions from data.

Envuelve una llamada al LLM

import { Guard, piiRedact, promptInjection, secrets, piiRestore, secretLeak } from "llm-io-guard";

const guard = new Guard({
  input: [piiRedact(), promptInjection(), secrets()],
  output: [piiRestore(), secretLeak()],
  onViolation: "block",
});

const inp = await guard.scanInput(textoDelUsuario);  // PII → vault, la PII no sale de tu proceso
if (!inp.allowed) throw new Error("entrada insegura");
const reply = await llamaATuLLM(inp.sanitized);      // manda el texto redactado
const out = await guard.scanOutput(reply, { vault: inp.vault }); // restaura la PII real
console.log(out.sanitized);

Defensa en capas, no una garantía. Las heurísticas tienen falsos negativos; complementa (no reemplaza) el mínimo privilegio y la separación de instrucciones y datos.
